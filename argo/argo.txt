Recommended architecture

A clean local setup looks like this:

GitHub repo 1: platform-gitops
    Argo CD Application manifests
    Kubernetes app manifests / Helm / Kustomize
    Argo WorkflowTemplates
kind cluster
    argocd namespace
    argo namespace for workflows
Argo CD
    watches GitHub repo
    syncs app/workflow manifests into cluster
Argo Workflows
    runs workflows from templates already synced by Argo CD
Optional:
    GitHub Actions calls Argo CD API/CLI
    GitHub webhook triggers Argo Workflow directly or through Argo Events
    Argo CD is built for declarative GitOps from Git repos, and Argo Workflows exposes events/webhook-based submission for workflows.

7) Connect Argo CD to GitHub
Public repo
No credentials needed. Just reference the GitHub HTTPS repo URL in the Application.
Private repo
Argo CD supports private repos over HTTPS or SSH, and also supports GitHub App credentials.
Option A: SSH key
% export YOUR_GITHUB_USER=serhioms
% argocd repo add git@github.com:$YOUR_GITHUB_USER/platform-gitops.git \
  --ssh-private-key-path ~/.ssh/id_ed25519
Repository 'git@github.com:serhioms/platform-gitops.git' added

% export YOUR_GITHUB_USER=serhioms
% export YOUR_GITHUB_TOKEN=YOUR_GITHUB_TOKEN
% argocd repo add https://github.com/YOUR_GITHUB_USER/platform-gitops.git \
   --username $YOUR_GITHUB_USER \
   --password $YOUR_GITHUB_TOKEN
